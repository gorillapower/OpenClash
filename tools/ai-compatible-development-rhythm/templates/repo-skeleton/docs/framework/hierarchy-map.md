# Hierarchy Map

## Default Mapping

- Roadmap layer: optional, replace only if the tracker/team needs one
- Planning layer: optional, replace only if grouping adds planning value
- Execution layer: replace with the repo's normal implementation unit
- Child execution split: optional, replace only if the tracker/team decomposes work this way
- Defect layer: `Bug`
- Decision layer: `Decision Doc`
- Architecture layer: `Architecture Doc`

## Rule

Keep the roles stable even if the local labels differ across trackers or teams.
Do not require every role for every change.
