# Tracker Guidance: GitHub

Use this overlay after tracker detection when the repo uses GitHub issues and PRs.

## Principle

Do not invent hierarchy that GitHub does not naturally need.

## Recommended Lean Default

- `Issue` as the normal execution unit
- labels for work class and risk where useful
- milestones, projects, or issue groups only when they materially improve planning
- avoid creating parent/child structures unless the team already uses them

## Authoring Guidance

When setup writes the first-pass framework:

- treat the issue as the default work item
- state any project-board or milestone usage explicitly
- do not force a faux Epic/Feature structure unless the repo already uses it
