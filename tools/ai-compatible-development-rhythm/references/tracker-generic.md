# Tracker Guidance: Generic

Use this overlay when the tracker is unknown, mixed, or deliberately generic.

## Principle

Keep the hierarchy role-based, not tool-branded.

## Recommended Lean Default

- one normal implementation unit
- one defect unit
- one optional planning parent
- one rare roadmap parent

## Authoring Guidance

When setup writes the first-pass framework:

- identify the default implementation unit clearly
- mark planning parents as optional unless the team needs them consistently
- avoid introducing extra levels that the team is unlikely to maintain
